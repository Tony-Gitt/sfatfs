# 补充
补充一些不知道的
#### 文件存在哪
- 正如 wsl 的存储一样，文件系统本质存在分区中  
  但子系统的文件系统就需要在主系统的某个路径中  
  以镜像文件的形式出现  
  文件在镜像系统中，但如果解析不适配  
  主系统无法解析文件系统，只有自己的程序可以
- 本质上，还需要适配硬件驱动层  
  linux 的硬件驱动层是系统的 api  
  stm32 上的是 hal 库，还需要调用这些 api  
  而不能直接参与读写内存 (风险太高了)
  linux 中本质上是 ext4 形式存储的  
  就是 img 文件是 ext4 格式的，  
  但其中内容实际上是一整个磁盘，可以二次解析  
  写 api 时需要一些文件操作的知识

#### BPB BIOS Parameter Block
- 相当于规则表，知道目录，文件在哪  
  挂载时可以检查，初始化与操作时根据表操作地址
